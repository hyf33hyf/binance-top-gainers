# 部署教程：让涨幅榜采集 24 小时云端运行

这套方案的原理：把"定时检测币安涨幅榜"这件事交给 GitHub 的免费服务器去做，
你的网页只负责从 GitHub 读取结果并展示。这样即使你的电脑关机，数据依然会按设定的时间间隔持续更新。

**全程免费，不需要信用卡，跟着下面的步骤一步步操作即可。预计花费时间：15-20分钟。**

---

## 第一步：注册 GitHub 账号

1. 打开浏览器访问 **https://github.com/signup**
2. 按提示填写：邮箱 → 密码 → 用户名（这个用户名之后要用到，记下来）
3. 完成邮箱验证（GitHub会发验证码到你邮箱，输入即可）
4. 注册完成后会自动登录到 GitHub 主页

> 如果你已经有 GitHub 账号，跳过这一步，直接登录即可。

---

## 第二步：创建一个新仓库（Repository）

1. 登录后，点击右上角的 **"+"** 号 → 选择 **"New repository"**
2. 填写仓库信息：
   - **Repository name**（仓库名）：填 `binance-top-gainers`（或你喜欢的名字，记下来）
   - **Public / Private**：选择 **Public**（必须是公开的，否则网页读取不到数据）
   - 其他选项不用动，下面的 "Add a README file" 可以勾选也可以不勾
3. 点击绿色的 **"Create repository"** 按钮

创建完成后，你会看到一个空仓库页面，地址类似：
```
https://github.com/你的用户名/binance-top-gainers
```

---

## 第三步：把项目文件上传到仓库

你现在收到的文件夹 `evan-top-gainers-cloud` 里包含这些内容：

```
evan-top-gainers-cloud/
├── .github/
│   └── workflows/
│       └── fetch-gainers.yml      ← 定时任务配置（核心文件）
├── data/
│   └── history.json                ← 数据存储文件（会被自动更新）
├── scripts/
│   └── fetch_top_gainers.py        ← 采集脚本（核心文件）
└── index.html                       ← 你看到的网页
```

**上传方法（网页操作，不需要命令行）：**

1. 回到你刚创建的空仓库页面
2. 找到 **"uploading an existing file"** 这个链接点击（或者点击 "Add file" → "Upload files"）
3. 把 `evan-top-gainers-cloud` 文件夹里的**所有文件和文件夹**，整个拖拽到网页的上传区域
   - 注意：要保持文件夹结构！把整个 `evan-top-gainers-cloud` 文件夹里的内容（不是文件夹本身）拖进去
   - 如果浏览器不支持拖拽整个文件夹，可以分批上传：先上传 `index.html`，再分别进入子文件夹上传里面的文件
4. 滚动到页面底部，在 "Commit changes" 那里点击绿色按钮 **"Commit changes"**

上传完成后，仓库里应该能看到 `.github`、`data`、`scripts` 三个文件夹和 `index.html` 文件。

---

## 第四步：修改 index.html 里的仓库信息

刚才上传的 `index.html` 里有两行占位符需要改成你自己的信息：

1. 在 GitHub 仓库页面，点击 `index.html` 文件
2. 点击右上角的铅笔图标 ✏️（编辑此文件）
3. 用 `Ctrl+F`（或浏览器内查找）找到这两行：
   ```javascript
   const GITHUB_USER = 'YOUR_GITHUB_USERNAME';
   const GITHUB_REPO  = 'YOUR_REPO_NAME';
   ```
4. 把 `YOUR_GITHUB_USERNAME` 改成你的 GitHub 用户名
5. 把 `YOUR_REPO_NAME` 改成你的仓库名（如果按教程用的是 `binance-top-gainers`，就填这个）

例如，如果你的用户名是 `evan2026`，仓库名是 `binance-top-gainers`，改完应该是：
```javascript
const GITHUB_USER = 'evan2026';
const GITHUB_REPO  = 'binance-top-gainers';
```

6. 滚动到页面底部，点击绿色的 **"Commit changes"** 保存

## 第五步：手动触发一次，测试是否正常运行

1. 在仓库页面，点击顶部的 **"Actions"** 标签
2. 左侧应该能看到一个叫 **"币安涨幅榜定时采集"** 的工作流，点击它
3. 右侧会出现 **"Run workflow"** 按钮（一个下拉框+按钮），点击展开后再点击绿色的 **"Run workflow"**
4. 等待几秒后刷新页面，会看到一条新的运行记录，状态从黄色圆圈（运行中）变成绿色对勾（成功）
   - 如果变成红色叉号（失败），点进去看日志，最常见原因是网络超时，重新点一次 "Run workflow" 再试一次即可
5. 运行成功后，回到仓库的 `data/history.json` 文件，刷新查看，应该能看到今天的日期和币种数据已经写进去了

## 第六步：打开你的网页

1. 把 `index.html` 在浏览器中打开（双击文件，或拖进浏览器窗口）
2. 顶部应该显示 **"☁️ 云端数据已连接"**，下方能看到刚才采集到的币种列表

如果显示"云端连接失败"，检查：
- 第四步里的用户名和仓库名是否填写正确（注意大小写、不要有多余空格）
- 仓库是否设置成了 **Public**（如果是 Private，网页读取不到）
- 等1-2分钟再刷新（GitHub 的文件分发网络偶尔有几秒延迟）

## 之后会自动发生什么

- GitHub Actions 会按照配置的时间间隔（默认每30分钟）自动运行采集脚本，**完全在云端进行，与你的电脑无关**
- 每次运行会把新数据提交回仓库的 `data/history.json`
- 你的网页只要打开着，每60秒会自动检查一次云端数据是否有更新；就算不开着，下次打开时也会读取到最新数据
- 你可以随时关机、合盖、断网，云端的定时任务不受影响

---

## 如何调整采集设置

打开仓库里的 `.github/workflows/fetch-gainers.yml` 文件，点编辑：

**改采集频率**（找到这一行）：
```yaml
- cron: '*/30 * * * *'
```
- 改成 `*/15 * * * *` → 每15分钟一次
- 改成 `0 * * * *` → 每小时一次

**改 TOP 数量 / 计价货币**（找到这两行）：
```yaml
env:
  TOP_N: '5'        # 改成 '3' 即可变成 TOP3
  QUOTE: 'USDT'      # 可选 'USDT' 或 'BTC'
```

改完后点击 "Commit changes" 保存，下次定时触发就会按新设置运行。

## 常见问题

**Q: GitHub Actions 真的完全免费吗？**
A: 公开仓库的 GitHub Actions 完全免费，没有用量限制问题（私有仓库每月有2000分钟免费额度，但我们用的是公开仓库）。

**Q: 定时任务会不会被 GitHub 自动停用？**
A: 如果仓库连续60天没有任何活动（包括 Actions 运行），GitHub 可能会自动暂停定时任务。但只要这个采集脚本一直在跑（每30分钟一次），就不会被判定为"无活动"，可以放心长期运行。

**Q: 我能不能把仓库设为 Private（私密）？**
A: 可以设置 Actions 部分，但 `raw.githubusercontent.com` 读取私有仓库文件需要额外的访问令牌（Token），网页端直接 fetch 会失败。如果你需要私密方案，告诉我，我可以提供加 Token 验证的版本。

**Q: cron 定时不是很准时，有时候会晚几分钟？**
A: 这是 GitHub Actions 的正常现象，免费层级的定时任务不保证分秒不差触发，通常会有0-10分钟的延迟，属于正常范围。
